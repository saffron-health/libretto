> Author a minimal workflow, run it, and inspect the result.

## Agent prompt

```text
Create and validate a first Libretto workflow in my project.

1) Create src/workflows/scrape-page.ts with a workflow named "scrape-page" that:
   - opens https://example.com
   - reads the page title
   - logs the title
   - returns { title }
2) Run: npx libretto run src/workflows/scrape-page.ts
3) If it fails, debug using a named session:
   - npx libretto open https://example.com --session debug-example
   - npx libretto snapshot --session debug-example
   - npx libretto exec --session debug-example "await page.url()"
4) Apply fixes and re-run until the workflow succeeds.
5) Summarize changes, run output, and any debugging steps used.

Keep changes minimal and limited to this workflow task.
```

## Manual

<Steps>
  <Step title="Create a workflow file">

Create `src/workflows/scrape-page.ts`:

```ts src/workflows/scrape-page.ts
import { workflow } from "libretto";

export default workflow("scrape-page", async ({ page }) => {
  await page.goto("https://example.com");
  const title = await page.title();
  console.log(`Page title: ${title}`);

  return { title };
});
```
  </Step>

  <Step title="Run it locally">

```bash theme={null}
npx libretto run src/workflows/scrape-page.ts
```
  </Step>

  <Step title="Inspect page state when needed">

When a step fails or behaves unexpectedly, open a named debug session first:

```bash theme={null}
npx libretto open https://example.com --session debug-example
```

Then inspect and prototype with the same session:

```bash theme={null}
npx libretto snapshot --session debug-example
npx libretto exec --session debug-example "await page.url()"
```

When the prototype works, move the logic back into the workflow file and run again.
  </Step>
</Steps>
