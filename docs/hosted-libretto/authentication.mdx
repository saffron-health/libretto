> Create or join a hosted organization, store a local session, and issue API keys for hosted runs.

### Before you start

Ask the Libretto team for your hosted API URL, then set it before the first command:

```bash theme={null}
export LIBRETTO_API_URL=<value-from-the-libretto-team>
```

The examples below use `npx libretto`. If `libretto` is already on your `PATH`, drop `npx`.

### Create a new user account and organization

Use `auth signup` when you are creating a brand new hosted organization:

```bash theme={null}
npx libretto experimental auth signup --api-url "$LIBRETTO_API_URL"
```

The CLI prompts for your name, email, password, organization name, organization slug, and the alert email that Hosted Libretto should use for hosted workflow failure notifications. After the account is created, it waits for email verification and stores the hosted session locally in `~/.libretto/auth.json`.

<Note>
  A Libretto user can only belong to one organization at a time. If your team already has an organization, use an invite instead of creating a second account.
</Note>

If the organization slug is already taken, the CLI re-prompts for just the slug and keeps the rest of your signup input.

### Log in to an existing account

Use `auth login` when the account already exists:

```bash theme={null}
npx libretto experimental auth login
```

This stores a new hosted session in `~/.libretto/auth.json`. If the account is not verified yet, the CLI re-sends the verification email.

To clear local credentials:

```bash theme={null}
npx libretto experimental auth logout
```

### Check the active identity

Use `whoami` to confirm which hosted account the CLI is using:

```bash theme={null}
npx libretto experimental auth whoami
```

If `LIBRETTO_API_KEY` is set, the CLI prefers that key over the stored session cookie.

### Invite a teammate

Use `invite` from an account that can manage members in the active organization:

```bash theme={null}
npx libretto experimental auth invite teammate@example.com --role member
```

The command prints the invitation id and the organization slug that the recipient needs for `accept-invite`.

### Accept an invite

Use `accept-invite` when you are joining an existing organization:

```bash theme={null}
npx libretto experimental auth accept-invite <tenant-slug> <invitation-id> --api-url "$LIBRETTO_API_URL"
```

If you are not logged in yet, the CLI creates the account first, waits for email verification, and then saves the hosted session locally.

If you are already logged in, `accept-invite` only works when that account is not already a member of another organization.

### Issue an API key

Use API keys for project `.env` files, CI, or other non-interactive hosted runs:

```bash theme={null}
npx libretto experimental auth api-key issue --label laptop-dev
```

The command prints the raw key once. The CLI does not store that key on disk for you.

Add it to your project:

```bash theme={null}
LIBRETTO_API_URL=https://api.libretto.dev
LIBRETTO_API_KEY=<issued-key>
```

To inspect existing keys:

```bash theme={null}
npx libretto experimental auth api-key list
```

To revoke a key:

```bash theme={null}
npx libretto experimental auth api-key revoke <key-id>
```

### Troubleshooting

- If the CLI says no hosted URL is configured, set `LIBRETTO_API_URL` or pass `--api-url` on the first auth command.
- If signup fails because the email already exists, log in to that account or delete the existing hosted account before retrying.
- If an invite fails for a logged-in user, make sure that user is not already a member of another Libretto organization.
